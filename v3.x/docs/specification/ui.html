一、名词统一译法

  • DPL(Design Pattern Libaray): 设计模式库。
  • 组件(Component)：DPL的组成部分，DPL 内的每个元素都称为组件，一个组件是一套功能的集合。一个组件内可以包含子组件。两个组件之间也可以互相依赖。
  • 控件(Control)：控件是专门负责界面功能的组件。控件本身是和业务无关的。
  • 模块(Module)：功能专一的组件。
  • 框架(Framework)：专门为解决一类目标而写的代码统称为框架。
  • 私有文件（夹）：开发组件时所使用的、不发布的文件（夹）叫私有文件（夹），私有文件（夹）在命名上以下划线（_）开头。
  • 临时文件（夹）：临时使用的或由程序生成的文件（夹），私有文件（夹）在命名上以波浪线（~）开头。

三、文件夹格式

  • assets: 组件所需用的静态资源文件。
    • scripts: 组件所需用的 Javascript 文件或其它脚本文件（如 .htc 文件）
    • images：组件所需用的图片、图标和多媒体文件。
    • styles: 组件所需用的 CSS 样式文件或其它样式文件（如 .theme 文件)
  • index.html: 组件的示例页。
如果一个组件包含多个子组件，可以在组件跟目录新建一个子组件名的 html 文件。并在 assets 相应位置新建同名文件。 如：添加一个 button 子组件，应该新建以下 3 个文件：
  • button.html
  • assets/scripts/button.js
  • assets/styles/button.css
如果项目中使用了 lesscss, 则结构为:
  • button.html
  • assets/scripts/button.js
  • assets/styles/button.less

文件和文件夹名字必须为全小写。

四、CSS 命名规范

为防止组件之间的冲突，必须对组件类命名作限制：

CSS 命名原则：

  1. 组件跟节点的 CSS 命名必须是 x-当前组件名。
  2. 组件内部节点的 CSS 命名必须以 x-当前组件名- 开头。
  3. 使用多个类名，而不是一个类多个功能。(即使用 .a-b 而不是使用 .a.b)
  4. 图片命名应和类名一致，不需要添加额外的 -bg 说明。如 textbox 组件的主背景图应该是 textbox.png 。

如 button 组件：

HTML 代码：

<a href="#"><span>文字</span></a>

正确的 CSS 命名：

<a href="#" class="x-button"><span class="x-button-content">文字</span></a>

错误的 CSS 命名：

<a href="#" class="x-button"><span class="x-content">文字</span></a> <a href="#" class="x-button-outer"><span class="x-button-content">文字</span></a> <a href="#" class="x-btn"><span class="x-btn-cnt">文字</span></a>

如 grid 组件：

HTML 代码：

<div>文字</div>

正确的 CSS 命名：

<div class="x-grid x-grid-2"></div>

错误的 CSS 命名：

<div class="x-grid-2"></div> <div class="x-grid x-2"></div>

以下列举了常见的 css 命名后缀。

  • -actived: 激活的、选中的、当前的。
  • -seleced: 选择的。
  • -highlight: 高亮的。
  • -plain: 简单版的、无边框的、删除背景的。
  • -collapsed: 折叠的、暂时关闭的。
  • -disabled: 禁用的。
  • -readonly: 只读的。
  • -alt: 可选的。

五、 CSS 样式值规范

z-index 规范

  • 101-200: 为了修复一些兼容性问题，而不得不用的 z-index。
  • 1000: 遮罩层。
  • 1001-2000: 遮罩层，弹窗级别，需要用户操作才能出现的浮层。
  • 10001-20000: 工具提示，泡泡提示，绝对顶层。
  • 其它范围供页面内使用，组件不得使用。

长度单位规范

全部使用 px 单位。以下情况使用特殊单位。

  • 大小为 0 的时候，不需要单位。
  • 统计文字数时，使用 em 单位。
  • 需要相对定位时，使用 %。

颜色值规范

全部使用 十六进制大写颜色值 ，允许缩写。

六、 Javascript 组件规范

  1. 对象名必须和组件名一样，且首字母大写。
  2. 内部传递原生 DOM 对象，而非通过 ID、CSS 选择器 或 其它方式。

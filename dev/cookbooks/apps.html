开发系统介绍

功能

开发系统就像是 jPlusUI 的后勤服务部，它让模块开发和使用更轻松。它提供：

  1. 管理所有模块，自动列出所有模块。查看生成的列表
  2. 新建/删除模块。
  3. 模块的演示和测试支持。（如本页面顶部的导航就是开发系统自动生成的）
  4. 其它开发工具。（点击右上角工具）

源码位置

开发系统源码在项目跟目录的 apps/ 目录。

模块演示和测试的支持

模块开发完成后，需要写一些文档和示例。开发系统帮助您快速写出漂亮的文档。

载入开发系统

只需引入 /apps/demo/demo.js 文件即可载入开发系统。

模块演示页面的基本结构

模块开发服务器

功能

模块源码本身不需要服务器支持，服务器主要用于模块开发和应用。功能有：

  1. 自动生成模块列表。扫描 /examples 文件夹里的页面，生成的列表缓存文件位于 /apps/data/modulelist.js。
  2. 模块打包、合并功能。库里有超过 200 个模块，项目中一般需要先提取需要使用的模块，打包合并。（具体见右上角 工具/模块打包工具）

启动服务器

双击 /apps/startserver.cmd 即可启动服务器。（非 Windows 系统需要先安装 node.js, 然后在控制台打开 /apps/startserver ）。

双击 /apps/start.cmd 也可启动服务器，并同时在浏览器打开服务器首页。

如果需要修改服务器默认端口，需要修改 /apps/data/server.json 和 /apps/demo/demo.js。
